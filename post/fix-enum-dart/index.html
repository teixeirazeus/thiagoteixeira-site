Arrumando a enumeração do dart

Dart é um linguagem de programação com uma sintaxe simples e uma estrutura de programação orientada a objetos. Gosto da sintexe do dart, algumas coisas lembram python e js, o que me deixa feliz. Porém, a enumeração é diferente…

Suponha que você precise enumerar um status de um pedido de compra. Em dart, você pode fazer isso como:

enum Status {
    PENDING,
    APPROVED,
    REJECTED,
}

Agora, você quer printar no console esses status.

print(Status.PENDING);
print(Status.APPROVED);
print(Status.REJECTED);

E o resultado é esse:

Status.PENDING
Status.APPROVED
Status.REJECTED

Ele vem com o nome do status, e não com o valor. Agora você pode fazer um split na string, utilizando o ponto como separador. Ou utilizar um map, para mapear o valor para o nome. O que não fica tão bonito.

print(Status.PENDING.toString().split(".").last);
print(Status.APPROVED.toString().split(".").last);
print(Status.REJECTED.toString().split(".").last);

O ideal seria colocar um .value, para acessar o valor. Como funciona em python.

Para isto, é precisso criar um metodo extendido para o enum.

extension StatusExtension on Status {
  String get value {
    return this.toString().split(".").last;
  }
}

Continua com o split, mas agora pode ser acessado por .value.

print(Status.PENDING.value);
print(Status.APPROVED.value);
print(Status.REJECTED.value);

Resultado:

PENDING
APPROVED
REJECTED

Ah, agora sim. Código limpo.
